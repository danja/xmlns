SPICE Vocabulary

SPICE (Simulation Program with Integrated Circuit Emphasis) is a general-purpose, open source analog electronic circuit simulator. It is a powerful program that is used in integrated circuit and board-level design to check the integrity of circuit designs and to predict circuit behavior.

The purpose of the SPICE Vocabulary is to enable SPICE models to be created and used in a Web-friendly form. The SPICE Vocabulary will be defined in RDF.

Resource List (editable)

The current plan is to create the RDF Vocabulary as a direct mapping of the SPICE notation language.

Motivation

Circuits can already be developed using Web technologies, as demonstrated by the remarkable Circuit Lab.

However traditional circuit development tends to be a fairly isolated affair, a given circuit being developed locally. This can potentially be augmented by using the distributed, social nature of the Web. Circuits are graph-shaped, and defined through nodes and connections between those nodes. The Web is a Giant Global Graph. Why not bring them together?

SPICE offers a standard notation for expressing circuit models and many tools can be used to simulate the circuits or assist in the development of circuit boards. The Resource Description Framework offers a standard way of expressing such models on the Web.

If circuits are expressed in RDF as linked data, then potentially very powerful search and discovery becomes available. As well as being able to search for circuits by name or description, the components used and circuit topologies are "visible" in the RDF model, and can be pattern-matched using the SPARQL query language.

Use Cases

purely informative

  • the Web as circuit repository using Linked Data discovery & search
  • Web-based simulation - decoupled from the circuit descriptions
  • distributed circuits : the I/O of one circuit make be connected to the I/O of any other circuit on the Web
  • collaborative development : possibly layered over existing distributed version control
  • social net : Facebook-like community infrastructure over the topic of electronics

Requirements

Vocabulary

  • basic modeling of circuit structure
  • vocabulary of core SPICE terms
  • extension model for additional terms
  • correspondences with existing vocabularies

Tools

  • SPICE notation to Turtle notation converter(s)
  • Turtle to SPICE converter(s)
  • testbed repository (online triplestore with UI optimised for circuits)
  • testbed simulator (server-side and/or client side, derived from existing tools)

Documentation

  • documentation for all the above
  • examples
  • guidelines for integration

:terminalCount "2"^^xsd:int ;

:spiceInstanceTemplate "R<name> <(+) node> <(-) node> [model name] <value>+ [TC = <TC1> [,<TC2>]]" ;

:spiceModelTemplate ".MODEL <model name> RES [model parameters]" ;

:spiceExample "RLOAD 15 0 2K" .
