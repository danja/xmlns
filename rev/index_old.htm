1 Introduction

This document provides a vocabulary specification for the purpose of creating sharable reviews and ratings of blogs, CDs, books, software, anything identifiable.

The vocabulary is suitable for inclusion in any RDF-compatible language : FOAF, RSS etc. This specification only defines the minimum terms needed to provide the review/rating capability, leaving the rest of the resource description to existing, more domain-appropriate vocabularies.

2. Namespaces

The XML namespace for this vocabulary is http://www.purl.org/stuff/rev# and the recommended prefix is review. i.e. in RDF/XML documents the following attribute should be included in the rdf:RDF element :

xmlns:review="http://www.purl.org/stuff/rev#"

Note: Reviews written using the RDF Review Vocabulary are also likely to use terms from the Dublin Core and FOAF namespaces :

xmlns:dc="http://purl.org/dc/elements/1.1/"
xmlns:foaf="http://xmlns.com/foaf/0.1/"

3. RDF Classes

Review

Refers to the review itself.

Associated Classes

foaf:Person

Used to describe the reviewer.

4. RDF Properties

hasReview

Used to say that a work (or other resource) has the review identified.

type

The media type (Book, Film, Music etc) will be given in a <review:type> element.

rating

The suggested value of the rating of the work will be an integer value 1 - 10, in a <review:rating> element scaled with 1 for Truly, Madly, Deeply and 10 for Terminator.

The application developer may choose a different scale, or whether to datatype this value (using RDF Datatypes).

Associated Properties

dc:creator

The relationship between the work and the creator of the work (author, director...) will usually be expressed using the Dublin Core vocabulary's <dc:creator> element.

5. Examples

5.1 Album Review

Snippet of a profile generated by FOAF-o-matic
(mm is the MusicBrainz namespace (http://musicbrainz.org/mm/mm-2.1#), rdf:nodeID="A0" points to a foaf:Person elsewhere in the file) 
   
  <mm:Album rdf:about="http://mm.musicbrainz.org/album/37b9a29b-2d39-441b-9ac6-81770916e5b5">
  
      <dc:title>Aftermath</dc:title>
  
      <review:hasReview>
          <review:Review>
              <review:rating>8</review:rating>
              <review:reviewer rdf:nodeID="A0"/>
              <dc:description>Classic.</dc:description>
          </review:Review>
      </review:hasReview>
  
      <dc:creator>
          <mm:Artist rdf:about="http://mm.musicbrainz.org/artist/b071f9fa-14b0-4217-8e97-eb41da73f598"/>
      </dc:creator>
  
  </mm:Album>
  
  <mm:Artist rdf:about="http://mm.musicbrainz.org/artist/b071f9fa-14b0-4217-8e97-eb41da73f598">
      <dc:title>The Rolling Stones</dc:title>
  </mm:Artist>

5.2 Review Embedded in RSS Feed

   <rdf:RDF xmlns:rdf="http://www.w3.org/1999/02/22-rdf-syntax-ns#"
                     xmlns:dc="http://purl.org/dc/elements/1.1/"
                     xmlns="http://www.purl.org/stuff/rev#"
                     xmlns="http://purl.org/rss/1.0/">

  <item rdf:about="http://mm.musicbrainz.org/album/bb4d834e-c21a-4288-bbee-d9b86c6c3f8b">

      <dc:title>Every Day</dc:title>
      <description></description>
      <dc:creator>
          <mm:Artist rdf:about="http://mm.musicbrainz.org/artist/7c158ea8-c0aa-410e-bdc1-20bba9759577">
              <dc:title>The Cinematic Orchestra</dc:title>
          </mm:Artist>
      </dc:creator>

     <review:hasReview>
          <review:Review>
              <review:rating>9</review:rating>
              <dc:description>Blinding</dc:description>
              <review:reviewer>
                  <foaf:Person>
                      <foaf:name>Leigh Dodds</foaf:name>
                  </foaf:Person>
          </review:Review>
      </review:hasReview>

  </item>

  </rdf:RDF>

Appendix A

RDF Schema

As a separate document : review.rdfs 

   
  <?xml version="1.0"?>
   
  <rdf:RDF xmlns:rdf="http://www.w3.org/1999/02/22-rdf-syntax-ns#"
                   xmlns:rdfs="http://www.w3.org/2000/01/rdf-schema#"
                   xmlns:foaf="http://xmlns.com/foaf/0.1/"
                   xmlns="http://www.purl.org/stuff/rev#"
                   xml:base="http://www.purl.org/stuff/rev#">

  <rdfs:Class rdf:ID="Review">
      <rdfs:comment>A review of an artistic
  work</rdfs:comment>
  </rdfs:Class>

  
  <rdf:Property rdf:ID="type">
      <rdfs:comment>The type of media of a work under
  review</rdfs:comment>
  </rdf:Property>

  <rdf:Property rdf:ID="rating">
      <rdfs:comment>A mark out of 10</rdfs:comment>
  </rdf:Property> 

  <rdf:Property rdf:ID="text">
      <rdfs:comment>The text of the review</rdfs:comment>
  </rdf:Property>

  <rdf:Property rdf:ID="hasReview">
      <rdfs:comment>Used to associate a work of art with a a
  review</rdfs:comment>
      <rdfs:domain
  rdf:resource="http://www.w3.org/2000/01/rdf-schema#Resource"/>
      <rdfs:range rdf:resource="#Review"/>
  </rdf:Property>

  <rdf:Property rdf:ID="reviewer">
      <rdfs:comment>The person that has written the
  review</rdfs:comment>
      <rdfs:domain rdf:resource="#Review"/>
      <rdfs:range rdf:resource="http://xmlns.com/foaf/0.1/Person"/> 
  </rdf:Property>
  </rdf:RDF>
